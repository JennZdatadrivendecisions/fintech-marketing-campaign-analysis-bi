# fintech-marketing-campaign-analysis-bi
本项目以某理财平台十周年庆营销活动为业务场景，展示了从需求定义、数据埋点、AI模拟造数到ETL数据清洗与BI可视化的完整数据分析工作流。

# 理财平台十周年大促：活动复盘与增长分析

## 项目概览

项目旨在通过构建可复用的活动分析框架与监控看板，深入洞察用户在“预热-引爆-返场”三阶段的转化路径，定位“下载-注册-首投”链路中的流失瓶颈，并为后续的精细化运营与数据驱动决策提供支持。

## 局限性

本项目采用“逆向工程”思路，目的是为了**展现商业数据分析思维与全栈技术落地能力**。受限于个人模拟数据的局限性，做以下说明：<br>
1. **数据仿真度局限：** 真实业务中复杂的用户行为（如频繁的跨端 Session 跳转、精准的末次触点归因细节等）难以通过纯代码完美还原，且生成的转化数字可能不完全符合真实商业环境。<br>
2. **分析重点声明：** 项目的核心价值在于 **“分析框架的设计”** 与 **“数据链路的打通”（Python造数 -> SQL处理 -> 星型模型构建 -> BI可视化）**。展示的是面对实际业务场景时，我将如何拆解指标、组织数据表关联，以及如何通过商业智能看板输出业务洞察。<br>

## 工作流程

1. 业务梳理: 基于 AARRR 模型，梳理从“曝光”到“投资”的完整转化路径，明确三阶段（预热期、引爆期、返场期）的运营策略与核心监控指标。<br>
2. 数据构建: 依据梳理好的数据字典，使用 Python 生成贴合实际业务逻辑的模拟数据（涵盖用户属性、点击流、游戏参与、订单交易等），通过 MySQL 脚本完成基础清洗、分层与宽表构建。<br>
3. 数据模型: 采用星型模型设计（详见 数据模型设计.xmind），以用户关键行为（注册、点击、参与、订单）为事实表，结合日期、用户属性、活动资源位等作为维度表，底层支撑多维分析需求。<br>
4. 分析呈现: 在 Power BI 中开发交互式看板，实现活动概览、流量转化漏斗、运营玩法效率评估及产品销售维度的可视化监控。<br>

## 核心业务洞察

1. 流量与转化瓶颈： 浮标与闪屏转化率突出 (最高达22%)，但主要流量入口的弹窗转化率未达预期 (17.99%)。新客链路中，“注册 -> 绑卡 -> 首投” 转化率逐级断崖式下跌，最终新客首投转化率仅为 6.7%，是核心流失节点。<br>
2. 游戏化运营效果： 抽奖与“攒财运”等轻量级互动完成率较好；但涉及社交裂变的“集福卡”玩法，随着难度增加（如邀请5人）用户流失严重，最终仅 1.6% 的用户合成卡片，说明裂变门槛设计过高。<br>
3. 客群与产品偏好： 核心投资贡献力集中在40-60岁的中老年客群（占比超50%）。用户整体呈现“稳健偏好”，R2/R3级别的中长期理财产品表现出明显的“高订单、高GMV”特征。<br>

## 如何运行本项目

- 下载 02_Data_Generation 中的 Python 脚本运行，生成基础 CSV 数据。<br>
- 将生成的 CSV 数据导入本地 MySQL 数据库中。<br>
- 运行 03_SQL_ETL 中的 SQL 脚本，完成数据清洗并生成后续 BI 所需的视图。<br>
- 使用 Power BI Desktop 打开 04_PowerBI_Dashboard/营销活动监控看板.pbix，并在数据源设置中更改为您本地的 MySQL 数据库连接即可查看交互式看板。<br>
